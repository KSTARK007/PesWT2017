Document Object Model(DOM)

Notes

  • The brower has a inbuilt which should follow a particular set of rules that says when the browser encounter the tag the browser will create a object of that tag
  • When ever we write a js to change the webpage the source code does not change, only the object are changed using DOM
  • if browser encounter the html tag it creates a object html and than moves on and then if it finds the head tag it creates a object and then as the head is nested in html it linkes the head tag and html tag and makes head a child node of html and soo on
  • if the browser encounter the closing tag then the link is nolonger done which ever tag comes later will not be linked to the prev parent node
  • API(application porgramming interface) can be a interface between the application and OS or one application to another
  • most of the methods we learn is applied on document object
